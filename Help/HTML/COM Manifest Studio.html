    1. Introduction

    Welcome to the Com Manifest Studio

    StSynch is a tool that helps with restructuring databases. It handles the DataFlex Embedded database, Pervasive.SQL and MS-SQL. Other databases that uses Data Access (DAW) drivers should also work, albeit not tested. If you do test with another database please let us know how it worked.


    How you use the tool


    1. At the office; Use it to take a snapshot of a development/source database and save it to a STS file. Check that the the Read Source Database and Write .sts file radio-button in the toolbar is active.


    1. In production; a) Use it to make a comparison between the snapshot (STS file) and the target database. b) Make an update of the target (customer) database. This will make the two databases structurally identical. Check that the Read .sts file and Update Target Database radio-button in the toolbar is active.


    An STS file is a file with an .sts extension and is used by the tool to save a snap-shot of a database. It is suggested to include the creation date with the file name. This makes it easier at customer site to know which .sts file should be used. In fact the tool will suggest todays date as part of the file name when it is created, but the file name can be changed to anything you like.
